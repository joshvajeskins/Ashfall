# Prompt 5: Pixel Art Assets

## Goal
Replace placeholder textures with proper pixel art sprites and tiles.

## Pre-Check: Review Issues
Before starting, check these issue docs:
- `docs/issues/ui/README.md` - Asset loading issues

## Context
Currently `BootScene.ts` generates placeholder textures programmatically:
- Colored rectangles for tiles
- Simple shapes for characters
- Basic colored squares for items

The game needs 16x16 or 32x32 pixel art in dark fantasy style.

## Tasks

### 5.2 Asset Generation Options

**Option A: AI Generation (Recommended for speed)**
Use DALL-E, Midjourney, or Stable Diffusion with prompts like:
- "16-bit pixel art warrior character, dark fantasy, transparent background, 32x32"
- "pixel art dungeon floor tile, stone, dark, seamless, 32x32"

**Option B: Free Asset Packs**
- itch.io has many free dungeon tileset packs
- OpenGameArt.org for public domain sprites
- Kenney.nl for CC0 assets

**Option C: Simple Pixel Art (Manual)**
Create minimal but distinct sprites using:
- Aseprite
- Piskel (free, web-based)
- Photoshop with pixel grid

### 5.4 Update DungeonScene Enemy Sprites

In `spawnEnemies()`, map enemy types to correct sprites:

```typescript
const ENEMY_SPRITES: Record<string, string> = {
  'Skeleton': 'enemy-skeleton',
  'Zombie': 'enemy-zombie',
  'Ghoul': 'enemy-ghoul',
  'Vampire': 'enemy-vampire',
  'Lich': 'enemy-lich',
  'Dungeon Lord': 'enemy-boss',
};

const spriteKey = ENEMY_SPRITES[enemy.name] || 'enemy-skeleton';
```

### 5.5 Update Player Sprite by Class

In `createPlayer()`:

```typescript
const CLASS_SPRITES: Record<string, string> = {
  'Warrior': 'player-warrior',
  'Rogue': 'player-rogue',
  'Mage': 'player-mage',
};

const spriteKey = CLASS_SPRITES[this.character.class] || 'player-warrior';
this.player = this.add.image(startX, startY, spriteKey).setDepth(10);
```

### 5.6 Keep Fallback Generation

Keep the placeholder generation as fallback if assets fail to load:

```typescript
private loadAssets(): void {
  this.load.on('loaderror', (file: Phaser.Loader.File) => {
    console.warn(`Failed to load: ${file.key}, using placeholder`);
    this.generatePlaceholderTexture(file.key);
  });

  // ... load actual assets
}
```

## Acceptance Criteria
- [ ] At least 6 distinct enemy sprites
- [ ] 3 player class sprites
- [ ] Tile set for dungeon (floor, wall, door, exit)
- [ ] Item icons for weapons, armor, potions
- [ ] No console errors about missing textures
- [ ] Game looks visually distinct from placeholder version

## Style Guidelines
- Dark fantasy theme (browns, grays, dark reds)
- Consistent pixel size (32x32 for characters/tiles)
- Transparent backgrounds on sprites
- High contrast for visibility
